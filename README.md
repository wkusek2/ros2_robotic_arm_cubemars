# ROS2 Robotic Arm 6-DOF

Educational project: a complete ROS2 Humble control stack for a 6-DOF robotic arm driven by CubeMars AK45-36 motors over CAN.

## Tech stack

- **ROS2 Humble**
- **C++17** — nodes, hardware interface, CAN communication
- **Python** — launch files
- **MoveIt2** — motion planning (integrated)
- **ros2_control** — hardware abstraction layer (integrated)

## Project structure

```
ros2_robotic_arm/
└── src/
    ├── arm6dof/
    │   ├── src/arm/
    │   │   ├── armnode.cpp              # ROS2 node: publishers only
    │   │   ├── ArmController.hpp/cpp    # Motor control: VESC CAN + MIT protocol
    │   │   ├── CanBridge.hpp/cpp        # Waveshare USB-CAN-A serial driver
    │   │   └── ArmHardwareInterface.hpp/cpp  # ros2_control SystemInterface plugin
    │   ├── msg/
    │   │   ├── MITFeedback.msg          # Single motor MIT feedback
    │   │   └── MITFeedbackArray.msg     # All motors MIT feedback
    │   ├── arm6dof_plugin.xml           # pluginlib registration
    │   ├── launch/main.launch.py        # Launches arm + full MoveIt2 stack
    │   └── urdf/arm6dof.urdf            # Robot description with ros2_control tags
    └── arm6dof_moveit_config/           # Generated by MoveIt Setup Assistant
        ├── config/                      # Kinematics, planners, controllers
        └── launch/                      # MoveIt2 launch files
```

## Architecture

```
MoveIt2 / RViz2
    └── JointTrajectoryController (ros2_control)
            └── ArmHardwareInterface  (ros2_control plugin)
                    └── ArmController
                            └── CanBridge → USB-CAN-A (2 Mbps) → AK45-36 motors

ArmNode (ROS2)
    ├── Publisher: joint_states
    ├── Publisher: arm/diagnostics
    └── Publisher: arm/mit_feedback
```

`ArmHardwareInterface` is the sole owner of `ArmController`. It runs inside the ros2_control `controller_manager` and is loaded automatically when the hardware interface plugin is registered.

## Topics

| Topic | Type | Direction | Description |
|-------|------|-----------|-------------|
| `joint_states` | `JointState` | pub | Position/velocity from servo feedback |
| `arm/diagnostics` | `DiagnosticArray` | pub | Motor controller temperature |
| `arm/mit_feedback` | `MITFeedbackArray` | pub | Full MIT feedback for all motors |

## Build and run

```bash
cd ~/ros2_robotic_arm
colcon build
source install/setup.bash
ros2 launch arm6dof main.launch.py
```

> The launch file starts the full MoveIt2 stack including RViz2. MoveIt2 motion planning requires the ros2_control `controller_manager` to load `ArmHardwareInterface`, which opens `/dev/ttyUSB0`.

## Motor commands (MIT mode, direct)

```bash
# Enable motor 1
ros2 topic pub --once /arm6dof/arm/mit_ctrl std_msgs/msg/Float64MultiArray "{data: [1, 1]}"

# Zero encoder on motor 1
ros2 topic pub --once /arm6dof/arm/mit_ctrl std_msgs/msg/Float64MultiArray "{data: [1, 2]}"

# Direct MIT command: motor 1, position=0 rad, kp=5, kd=0.5
ros2 topic pub --once /arm6dof/arm/mit_hw_cmd std_msgs/msg/Float64MultiArray "{data: [1, 0.0, 0.0, 5.0, 0.5, 0.0]}"

# Monitor MIT feedback
ros2 topic echo /arm6dof/arm/mit_feedback
```

## Hardware

- **Waveshare USB-CAN-A** (CH340, binary serial framing, 2 Mbps, `/dev/ttyUSB*`)
- **CubeMars AK45-36** motors (VESC firmware + MIT Cheetah mode)
  - POLE_PAIRS=14, GEAR_RATIO=36, Kt=2.009 Nm/A
  - MIT ranges: position ±12.5 rad (16-bit), velocity ±50 rad/s (12-bit), torque ±18 Nm (12-bit)
- **CAN protocol:**
  - Servo mode (VESC): extended 29-bit frames, `CMD_SET_POS`=4, `CMD_SET_CURRENT`=1, feedback at 0x29xx ~150 Hz
  - MIT mode: standard 11-bit frames, 8-byte packed float→uint fields
